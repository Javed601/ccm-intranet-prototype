SPFx Implementation Guide

How to port any design concept into a SharePoint Framework web part — click to expand

A · B · C · DGradient Card Layouts
  • Create a React SPFx web part. One LocationsGrid.tsx component, map over locations array returned by PnP.
  • Store data in a SharePoint List CCM_Locations — see schema below.
  • Fetch with @pnp/sp: sp.web.lists.getByTitle('CCM_Locations').items.getAll()
  • Gradients stored as two columns (GradientFrom, GradientTo) and interpolated in JSX: style={{background: `linear-gradient(135deg,${item.GradientFrom},${item.GradientTo})`}}
  • CSS custom properties (--primary, etc.) work natively in SPFx CSS Modules — wrap in :global() if needed.
E · FSplit Panel & Data Table
  • Option E sidebar totals: compute from fetched array using Array.reduce inside useMemo.
  • Option F: use Fluent UI DetailsList component for built-in sort/filter — maps directly to SharePoint data.
  • Country filter: useState<string>('all') + items.filter(i => filter === 'all' || i.CountryCode === filter).
  • Left-border accents: define a countryStyles map object, apply via className={styles[item.CountryCode]}.
GExplorer Panel
  • Two sibling components: <LocationList /> and <LocationPreview />, lift selectedId state to parent.
  • Pass onSelect(id) and selectedId as props — no state management library needed.
  • Preview transitions: CSS transition: background 0.4s on the header div — browser handles the animation.
  • Deep-link CTA: use this.context.pageContext.web.absoluteUrl + '/SitePage' for site page links.
  • Responsive: collapse list to a horizontal tabs on mobile using a CSS @media query.
HDark Typography Grid
  • Purely CSS-driven — no JavaScript needed in the component beyond rendering the list.
  • The ghosted background city text uses opacity: 0.04 on a ::before pseudo-element with content: attr(data-city).
  • SPFx: set the web part theme to dark/custom via ThemeProvider if needed, or override background inline.
  • Accent top bar uses inline style with gradient from list columns — no hardcoded colours in the component.
IPortrait Carousel
  • Attach a useRef to the scroll track div. Arrow buttons call trackRef.current.scrollBy().
  • Use IntersectionObserver on each card (threshold: 0.6) to detect the active card and update dot state.
  • CSS scroll-snap-type: x mandatory and scroll-snap-align: start handle the snapping — no JS snap logic needed.
  • Note: CSS scroll-snap is not supported in IE11. For SharePoint modern pages this is fine.
JEditorial Hero + Ticker
  • Set "isFullWidth": true in the web part manifest JSON for an edge-to-edge editorial look.
  • Ticker: duplicate the items array in JSX ([...items, ...items]) and animate with @keyframes translateX(-50%) — no JS needed.
  • Pause on hover: CSS animation-play-state: paused on :hover on the inner container.
  • The floating flag animation (jFloat) requires no transform on ancestor elements — check your SPFx zone wrapper.
  • Hero content (HQ data) can be hardcoded or fetched from a single filtered list item where IsHQ = true.
SharePoint List Schema — CCM_Locations
// SharePoint List: CCM_Locations (visible to all authenticated users) const listColumns = { Title: "Single line of text", // City display name Country: "Choice", // Germany | Netherlands | United Kingdom | Romania CountryCode: "Single line of text", // DE | NL | UK | RO FlagEmoji: "Single line of text", // 🇩🇪 — stored as Unicode emoji string SiteType: "Choice", // HQ | Manufacturing | Distribution | Operations Employees: "Number", GradientFrom: "Single line of text", // e.g. #1a237e GradientTo: "Single line of text", // e.g. #5c6bc0 SitePageURL: "Hyperlink", // deep-link to site intranet page IsHQ: "Yes/No", SortOrder: "Number", // controls display sequence Description: "Multiple lines of text" // used in Option G preview panel };
